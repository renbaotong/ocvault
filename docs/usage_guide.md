# 公司债募集说明书知识库 - 使用指南

## 快速开始

### 1. 处理新募集说明书

```bash
# 将 PDF 放入 raw/ 目录
cp new_prospectus.pdf raw/

# 运行提取脚本
python3 scripts/extract_prospectus.py
```

### 2. 在 Obsidian 中打开

```bash
# 用 Obsidian 打开 knowledge 目录
open -a Obsidian knowledge/
```

## 目录结构

```
ocvault/
├── raw/                    # 原始 PDF 文件
├── knowledge/              # Obsidian 知识库
│   ├── 00-Meta/           # 索引文件
│   ├── 01-发行条款/       # 债券发行条款
│   ├── 02-募集资金运用/   # 资金用途
│   ├── 03-发行人基本情况/ # 发行人信息
│   ├── 04-主营业务分析/   # 主营业务
│   ├── 05-财务状况/       # 财务分析
│   ├── 06-区域分析/       # 区域经济财政
│   └── 07-模板参考/       # Obsidian 模板
└── scripts/               # 处理脚本
```

## 提取字段说明

### 第二节 发行条款
**自动提取字段：**
- 发行人全称
- 债券全称/简称
- 注册规模
- 本期发行规模
- 债券期限
- 增信措施（担保机构）
- 主体评级
- 债项评级

**需手动补充：**
- 票面利率（部分 PDF 格式差异）
- 发行日期（具体年月日）
- 特殊条款（回售、赎回等）

### 第三节 募集资金运用
- 募集资金总额
- 资金使用计划
- 项目建设类（合法性文件）
- 偿还贷款类

### 第四节 发行人基本情况
- 发行人概况
- 历史沿革
- 股权结构
- 重要权益投资
- 主营业务（收入、毛利、毛利率）

### 第五节 财务状况
- 主要财务数据
- 经营性/非经营性资产
- 政府性资产识别
- 偿债能力指标

## 标签体系

- `#债券类型`: `#公司债` `#乡村振兴债` `#革命老区债`

## 批量处理

```bash
# 处理 raw/ 目录下所有 PDF
python3 scripts/extract_prospectus.py

# 查看生成的笔记数量
find knowledge -name "*.md" | wc -l
```

## Camelot 表格提取

```bash
# 使用 camelot-py 提取 PDF 中的表格数据
python3 scripts/extract_tables_camelot.py
```

### 提取结果

表格数据追加到 `knowledge/05-财务状况/{发行人}-财务分析.md` 文件末尾：

- **原始财务数据表** - 募集说明书第五节提取的财务报表
- **募集资金用途表** - 募集说明书第三节提取的项目投资明细

### 支持的表格类型

1. **财务报表** - 资产负债表、利润表、现金流量表
2. **财务指标表** - 偿债能力、营运能力指标
3. **募集资金用途表** - 项目投资明细、资金使用计划
4. **子公司财务表** - 重要权益投资财务数据

### 数据质量

- 自动识别财务相关表格（基于关键词匹配）
- 转换为标准 Markdown 表格格式
- 保留原始页码信息便于核对

### 配置环境变量

```bash
# 在 ~/.zshrc 或 ~/.bashrc 中添加
export TUSHARE_TOKEN=your_token
```

### 同步宏观经济和债券数据

```bash
# 运行 Tushare 数据同步脚本
python3 scripts/tushare_data_sync.py
```

### 数据覆盖

- **宏观经济**: CPI、PPI、PMI、GDP、货币供应量 (M0/M1/M2)
- **利率数据**: Shibor、LPR、国债收益率曲线
- **社会融资**: 社融增量月度数据
- **债券市场**: 可转债基础信息、债券回购利率

### 数据更新频率

建议每周或每月运行一次同步脚本，保持数据最新。

## 注意事项

1. **PDF 格式**：确保 PDF 为文本可选格式（非扫描图片）
2. **数据校验**：提取的数据需要人工校验，特别是财务数字
3. **增量更新**：新 PDF 放入 raw/ 后重新运行脚本即可
4. **Git 备份**：已配置 git，可定期 commit 备份

## 常见问题

**Q: 提取的数据不准确？**
A: PDF 格式差异可能导致提取偏差，建议人工核对关键字段

**Q: 如何修改模板？**
A: 编辑 `knowledge/01-发行条款/_template.md` 等模板文件
